Aarogyam — Low-Fidelity Wireframe Package

"One Health Identity For Life." A digital health-identity platform giving every citizen a unique Aarogyam ID that centralises medical reports, prescriptions, diagnoses, a health timeline, doctor consultations, and a QR health card. This document is a grayscale, production-oriented wireframe system covering 23 screens across three roles — Patient, Doctor, Admin — plus flows, responsive frames, a component library, and state designs. Built for direct hand-off to Figma and React.

How to read these wireframes: solid dark boxes = primary actions; outlined boxes = secondary; gray bars = text/content not yet written; boxes with an ✕ = image/illustration placeholders; checkerboard squares = QR codes. Nothing here is final visual design — it defines layout, hierarchy, components, and flow.

Role-Based Access Flow

↑ top

A single login resolves to a role claim. The router guards each zone; unauthorised access falls back to the user's own home or an error state.

CapabilityPatientDoctor (approved)Admin

Gate logic: isAuthed → hasRole(route.role) → (role==='doctor' ? isApproved : true). A pending doctor is authenticated but routed only to the Approval Pending screen until an admin approves.
